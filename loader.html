Metal Gear Solid V: The Phantom Pain technical information

As a modder, I restart game a lot of times and packing lua files back takes away my free time. That's why I had to make a loader, which reads lua scripts situated outside *.dat archives. It also is capable of binding functions used in external lua files to keys. Keys can be defined in a separate config file, stored outside of archives as well. The only thing you'll ever need to pack is the loader itself (and assets if you use them in your mod).

Installation

Place LINK TO LOADER into 00.dat, /Assets/tpp/script/lib/scriptloader.lua - drop it here. Add ≶Entry FilePath="/Assets/tpp/script/lib/scriptloader.lua" Compressed="false" /&rg; into your 00.dat.xml (if you use GzsTool). If you use MGS_QAR_TOOL, add "STRING" to your 00.inf. Open 00\Assets\tpp\script\lib\Tpp.lua and find "/Assets/tpp/script/lib/TppMbFreeDemo.lua". Add after ', "/Assets/tpp/script/lib/scriptloader.lua" (yes, with a comma). Open 00\Assets\tpp\script\lib\TppMain.lua and find "TppMission.UpdateForMissionLoad". Add after ',DropWeapon.Update' (with comma as well). Pack your 00.dat.

Usage

Script loader will look for your lua files in different directories, I use main game directory (SteamApps\common\MGS_TPP\lua). You will need to create 'lua' folder there and put your scripts there along with provided config. Lua scripts have to be listed in config like this: EXAMPLE. By pressing MODKEY+R loader will reload all scripts mentioned in config.
